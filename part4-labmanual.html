Experiment 1Installing Flutter & Dart, and Dart Language Basics

Aim: (a) To install the Flutter SDK and Dart SDK and verify the setup. (b) To write a simple Dart program to understand core language basics — variables, data types, operators, conditionals, loops, and functions.

Software Required: A computer with internet access, Flutter SDK (bundled with Dart SDK), Visual Studio Code or Android Studio with the Flutter/Dart plugins, an Android Emulator or physical Android device with USB debugging enabled.

Part (a): Installing Flutter and Dart SDK — Procedure

  1. Download the Flutter SDK zip file for your operating system from flutter.dev and extract it to a permanent folder (e.g. C:\src\flutter on Windows, ~/development/flutter on macOS/Linux).
  2. Add the flutter/bin folder to your system's PATH environment variable, so the flutter and dart commands work from any terminal location.
  3. Install Android Studio (for the Android SDK and an emulator) and, inside it, install the Flutter and Dart plugins from the plugin marketplace.
  4. Open a terminal and run flutter doctor — this checks your entire setup and lists anything still missing.
  5. Resolve any issues flutter doctor reports (accepting Android licenses with flutter doctor --android-licenses is a common one), then run flutter doctor again until all checks pass.
  6. Create a test project with flutter create test_app, then run it with flutter run to confirm everything works end-to-end.

Part (b): Program — Dart Language Basics

// Experiment 1(b) - Dart Language Basics
// This program demonstrates variables, data types, operators,
// conditionals, loops, lists, and functions in Dart.

void main() {
  // ---------- VARIABLES & DATA TYPES ----------
  String studentName = 'Sudhakar';   // String - holds text
  int rollNumber = 1;                // int - holds whole numbers
  double cgpa = 9.2;                 // double - holds decimal numbers
  bool isPassed = true;               // bool - holds true/false

  // Print the basic details using string interpolation ($variable)
  print('Student Name : $studentName');
  print('Roll Number  : $rollNumber');
  print('CGPA         : $cgpa');
  print('Passed?      : $isPassed');
  print('----------------------------------');

  // ---------- OPERATORS ----------
  int marks1 = 85;                    // First subject's marks
  int marks2 = 90;                    // Second subject's marks
  int totalMarks = marks1 + marks2;   // '+' is the addition operator
  double average = totalMarks / 2;    // '/' is the division operator

  print('Total Marks  : $totalMarks');
  print('Average      : $average');
  print('----------------------------------');

  // ---------- CONDITIONAL STATEMENT (if-else) ----------
  String grade;                       // Will hold the computed grade
  if (average >= 90) {
    grade = 'A+';                      // Runs only if average is 90 or above
  } else if (average >= 75) {
    grade = 'A';                       // Runs if average is between 75 and 89
  } else {
    grade = 'B';                       // Runs for anything below 75
  }
  print('Grade        : $grade');
  print('----------------------------------');

  // ---------- LOOP (for) + LIST ----------
  List<int> subjectMarks = [85, 90, 88]; // A list holding 3 subjects' marks
  for (int i = 0; i < subjectMarks.length; i++) {
    // i starts at 0 and runs while i is less than the list's length
    print('Subject ${i + 1} Marks: ${subjectMarks[i]}');
  }
  print('----------------------------------');

  // ---------- FUNCTION CALL ----------
  String message = greetStudent(studentName); // Call the function below
  print(message);

  // ---------- ENCOURAGING FINAL OUTPUT ----------
  print('----------------------------------');
  print('🎉 Great job! You just ran your first Dart program successfully!');
}

// A function that takes a String parameter and RETURNS a String
String greetStudent(String name) {
  return 'Welcome to Dart programming, $name!'; // 'return' sends this value back
}

Experiment 3Responsive UI with MediaQuery and Breakpoints

Aim: (a) To design a responsive UI that adapts to different screen sizes. (b) To implement media queries and breakpoints for responsiveness.

Software Required: Flutter SDK, VS Code/Android Studio, an emulator (test by resizing the emulator window or using the responsive preview in DevTools).

Part (a): Program — Responsive UI using MediaQuery

// Experiment 3(a) - Responsive UI using MediaQuery
import 'package:flutter/material.dart';

void main() {
  runApp(const ResponsiveApp());
}

class ResponsiveApp extends StatelessWidget {
  const ResponsiveApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      home: const ResponsiveHome(),
    );
  }
}

class ResponsiveHome extends StatelessWidget {
  const ResponsiveHome({super.key});

  @override
  Widget build(BuildContext context) {
    // MediaQuery.of(context) gives details about the whole screen
    double screenWidth = MediaQuery.of(context).size.width; // Read screen width

    bool isTablet = screenWidth > 600; // true if width is over 600 pixels

    return Scaffold(
      appBar: AppBar(title: const Text('Responsive UI with MediaQuery')),
      body: Center(
        // Show a different layout depending on the detected screen width
        child: isTablet ? buildTabletLayout(screenWidth) : buildMobileLayout(screenWidth),
      ),
    );
  }

  // Layout used on narrow (phone) screens
  Widget buildMobileLayout(double width) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        const Icon(Icons.phone_android, size: 80, color: Colors.indigo),
        const SizedBox(height: 12),
        Text('Mobile Layout (width: ${width.toInt()}px)'),
        const SizedBox(height: 12),
        const Text('🎉 Great! The UI adapted to a small screen!'),
      ],
    );
  }

  // Layout used on wide (tablet) screens
  Widget buildTabletLayout(double width) {
    return Row(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        const Icon(Icons.tablet_mac, size: 80, color: Colors.teal),
        const SizedBox(width: 20),
        Column(
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            Text('Tablet Layout (width: ${width.toInt()}px)'),
            const SizedBox(height: 12),
            const Text('🎉 Great! The UI adapted to a large screen!'),
          ],
        ),
      ],
    );
  }
}

Expected Output

On a phone-sized screen, a vertical layout appears with a large phone icon, the detected width in pixels, and a green confirmation message. On a tablet-sized screen (or a resized emulator window wider than 600px), a horizontal layout appears instead, with a tablet icon beside the text.

Part (b): Program — Breakpoints using LayoutBuilder

// Experiment 3(b) - Breakpoints using LayoutBuilder
import 'package:flutter/material.dart';

void main() {
  runApp(const BreakpointApp());
}

class BreakpointApp extends StatelessWidget {
  const BreakpointApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      home: Scaffold(
        appBar: AppBar(title: const Text('Breakpoints with LayoutBuilder')),
        // LayoutBuilder reports the space available to THIS widget
        body: LayoutBuilder(
          builder: (context, constraints) {
            // Three breakpoints: mobile < 600, tablet < 1024, else desktop
            if (constraints.maxWidth < 600) {
              return buildLayout('Mobile', 1, Colors.red);      // 1 column
            } else if (constraints.maxWidth < 1024) {
              return buildLayout('Tablet', 2, Colors.orange);   // 2 columns
            } else {
              return buildLayout('Desktop', 4, Colors.green);   // 4 columns
            }
          },
        ),
      ),
    );
  }

  // Builds a simple grid with the given number of columns
  Widget buildLayout(String label, int columns, Color color) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        Text('$label Breakpoint Detected', style: const TextStyle(fontSize: 20)),
        const SizedBox(height: 12),
        GridView.count(
          shrinkWrap: true,          // Let the grid size itself to its content
          crossAxisCount: columns,   // Number of columns based on breakpoint
          padding: const EdgeInsets.all(16),
          children: List.generate(
            columns * 2, // Create a few sample boxes to fill the grid
            (index) => Container(
              margin: const EdgeInsets.all(4),
              color: color.withOpacity(0.3),
              height: 60,
              child: Center(child: Text('Box ${index + 1}')),
            ),
          ),
        ),
        const SizedBox(height: 12),
        const Text('🎉 Well done! You implemented responsive breakpoints!'),
      ],
    );
  }
}

Expected Output

On a narrow window, a single red-tinted column of boxes appears under the label "Mobile Breakpoint Detected". Resizing the window past 600px switches to two orange columns labelled "Tablet Breakpoint Detected"; past 1024px, four green columns appear labelled "Desktop Breakpoint Detected" — all updating live as the window is resized.

KEY POINTS
  • Result: A responsive UI adapting to different screen sizes was designed using MediaQuery, and breakpoint-based layouts were implemented using LayoutBuilder, both executed successfully.

  Widget build(BuildContext context) {

            const Text('🎉 Great! You navigated successfully!'),

                // Navigator.pop REMOVES the current screen, showing the previous one

              },

          ],
        ),
      ),
    );
  }

// Experiment 5(a) - Stateless vs Stateful Widgets
import 'package:flutter/material.dart';

void main() {
  runApp(const StateDemoApp());
}

class StateDemoApp extends StatelessWidget {
  const StateDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      home: Scaffold(
        appBar: AppBar(title: const Text('Stateless vs Stateful')),
        body: const Column(
          children: [
            SizedBox(height: 20),
            GreetingWidget(name: 'Sudhakar'), // A StatelessWidget
            SizedBox(height: 40),
            CounterWidget(), // A StatefulWidget
          ],
        ),
      ),
    );
  }
}

// StatelessWidget - output depends ONLY on the constructor arguments
class GreetingWidget extends StatelessWidget {
  final String name; // Set once, never changes
  const GreetingWidget({super.key, required this.name});

  @override
  Widget build(BuildContext context) {
    return Text('Hello, $name! (StatelessWidget)', style: const TextStyle(fontSize: 18));
  }
}

// StatefulWidget - CAN hold data that changes over time
class CounterWidget extends StatefulWidget {
  const CounterWidget({super.key});

  @override
  State<CounterWidget> createState() => _CounterWidgetState();
}

// The State object holds the mutable data for CounterWidget
class _CounterWidgetState extends State<CounterWidget> {
  int _count = 0; // Mutable field, starts at 0

  // Increases the counter and tells Flutter to redraw the widget
  void _increment() {
    setState(() {
      _count++; // Must happen INSIDE setState so Flutter knows to rebuild
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text('Counter Value: $_count', style: const TextStyle(fontSize: 20)),
        const SizedBox(height: 8),
        ElevatedButton(onPressed: _increment, child: const Text('Increment')),
        const SizedBox(height: 8),
        if (_count >= 5) // Encouragement appears after enough taps
          const Text('🎉 Great! You understood StatefulWidgets!',
              style: TextStyle(color: Colors.green)),
      ],
    );
  }
}

Expected Output

The screen shows "Hello, Sudhakar! (StatelessWidget)" at the top, followed by a live counter starting at 0 with an "Increment" button. Each tap increases the number shown; after 5 taps, a green message confirms understanding of StatefulWidget behaviour.

Part (b): Program — State Management with setState and Provider

// Experiment 5(b) - State Management with setState and Provider
import 'package:flutter/material.dart';
import 'package:provider/provider.dart'; // From the provider package

void main() {
  runApp(
    // ChangeNotifierProvider makes CounterModel available to all widgets below it
    ChangeNotifierProvider(
      create: (context) => CounterModel(),
      child: const ProviderDemoApp(),
    ),
  );
}

// CounterModel holds the shared count and notifies listeners when it changes
class CounterModel extends ChangeNotifier {
  int _count = 0;             // Private - only this class can modify it
  int get count => _count;    // Public getter - other widgets read through this

  void increment() {
    _count++;                 // Update the private field
    notifyListeners();        // Tell every watching widget to rebuild
  }
}

class ProviderDemoApp extends StatelessWidget {
  const ProviderDemoApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      home: Scaffold(
        appBar: AppBar(title: const Text('setState vs Provider')),
        body: const Center(child: CounterDisplay()),
      ),
    );
  }
}

// This widget only DISPLAYS the count and rebuilds automatically when it changes
class CounterDisplay extends StatelessWidget {
  const CounterDisplay({super.key});

  @override
  Widget build(BuildContext context) {
    // context.watch SUBSCRIBES this widget to CounterModel's changes
    final count = context.watch<CounterModel>().count;

    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        Text('Provider Counter: $count', style: const TextStyle(fontSize: 22)),

// SingleTickerProviderStateMixin supplies the 'vsync' AnimationController needs
class _FadeAnimationAppState extends State<FadeAnimationApp>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;   // Controls the animation's timing
  late Animation<double> _fadeAnimation;   // The value that changes over time

  @override
  void initState() {
    super.initState();
    // Create a controller that runs its animation over 2 seconds
    _controller = AnimationController(
      vsync: this,
      duration: const Duration(seconds: 2),
    );
    // Tween defines start (0.0 = invisible) and end (1.0 = fully visible) values
    _fadeAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(_controller);
  }

  @override
  void dispose() {
    _controller.dispose(); // Always release the controller's resources
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      home: Scaffold(
        appBar: AppBar(title: const Text('Fade Animation')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              // FadeTransition automatically animates its child's opacity
              FadeTransition(
                opacity: _fadeAnimation,
                child: const Icon(Icons.flutter_dash, size: 100, color: Colors.blue),
              ),
              const SizedBox(height: 24),
              ElevatedButton(
                onPressed: () {
                  _controller.forward(from: 0); // Play the animation from the start
                },
                child: const Text('Fade In'),
              ),
              const SizedBox(height: 12),
              const Text('🎉 Nice! You created your first Flutter animation!'),
            ],

void main() {

class ApiFetchApp extends StatelessWidget {

  @override
  Widget build(BuildContext context) {

      home: const FetchDataScreen(),

// A simple model class representing one user from the API
class ApiUser {
  final String name;
  final String email;

  ApiUser({required this.name, required this.email});

  // Factory constructor - builds an ApiUser from a JSON map
  factory ApiUser.fromJson(Map<String, dynamic> json) {
    return ApiUser(
      name: json['name'] as String,
      email: json['email'] as String,
    );
  }
}

// Fetches a list of users from a free public test API
Future<List<ApiUser>> fetchUsers() async {
  final response = await http.get(
    Uri.parse('https://jsonplaceholder.typicode.com/users'), // Public test API
  );

  if (response.statusCode == 200) {
    // Decode the JSON response body into a Dart List
    List<dynamic> data = jsonDecode(response.body);
    // Convert each JSON object into an ApiUser instance
    return data.map((json) => ApiUser.fromJson(json)).toList();
  } else {
    throw Exception('Failed to load users'); // Thrown on a server error
  }
}

class FetchDataScreen extends StatefulWidget {
  const FetchDataScreen({super.key});

  @override
  State<FetchDataScreen> createState() => _FetchDataScreenState();
}

class _FetchDataScreenState extends State<FetchDataScreen> {
  late Future<List<ApiUser>> _usersFuture; // Holds the pending/completed call

  @override
  void initState() {
    super.initState();
    _usersFuture = fetchUsers(); // Start the API call once, when the screen loads
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Fetch Data from REST API')),
      body: FutureBuilder<List<ApiUser>>(
        future: _usersFuture,
        builder: (context, snapshot) {
          // While waiting for the response, show a loading spinner
          if (snapshot.connectionState == ConnectionState.waiting) {
            return const Center(child: CircularProgressIndicator());
          }
          // If something went wrong, display the error
          if (snapshot.hasError) {
            return Center(child: Text('Error: ${snapshot.error}'));
          }
          // Data has arrived successfully
          final users = snapshot.data!;
          return Column(
            children: [
              const Padding(
                padding: EdgeInsets.all(12),
                child: Text('🎉 Great! Data fetched successfully from the API!'),
              ),
              Expanded(

Part (b): Program — Displaying Fetched Data Meaningfully (Cards + Pull-to-Refresh)

// Experiment 10(b) - Displaying Fetched Data in a Meaningful, Styled Way
import 'package:flutter/material.dart';
import 'dart:convert';
import 'package:http/http.dart' as http;

void main() {
  runApp(const MeaningfulDisplayApp());
}

class MeaningfulDisplayApp extends StatelessWidget {
  const MeaningfulDisplayApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      debugShowCheckedModeBanner: false,
      home: const PostsScreen(),
    );
  }
}

// Model representing one blog post from the API
class Post {
  final String title;
  final String body;

  Post({required this.title, required this.body});

  factory Post.fromJson(Map<String, dynamic> json) {
    return Post(
      title: json['title'] as String,
      body: json['body'] as String,
    );
  }
}

// Fetches the first 10 posts from the public test API
Future<List<Post>> fetchPosts() async {
  final response = await http.get(
    Uri.parse('https://jsonplaceholder.typicode.com/posts?_limit=10'),
  );
  if (response.statusCode == 200) {
    List<dynamic> data = jsonDecode(response.body);
    return data.map((json) => Post.fromJson(json)).toList();
  } else {
    throw Exception('Failed to load posts');
  }
}

class PostsScreen extends StatefulWidget {
  const PostsScreen({super.key});

  @override
  State<PostsScreen> createState() => _PostsScreenState();
}

class _PostsScreenState extends State<PostsScreen> {
  late Future<List<Post>> _postsFuture;

  @override
  void initState() {
    super.initState();
    _postsFuture = fetchPosts(); // Load the posts once when the screen opens
  }

  // Re-fetches the posts - called when the user pulls down to refresh
  Future<void> _refresh() async {
    setState(() {
      _postsFuture = fetchPosts(); // Trigger a fresh API call
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Blog Posts (Styled Display)')),
      body: FutureBuilder<List<Post>>(
        future: _postsFuture,
        builder: (context, snapshot) {
          if (snapshot.connectionState == ConnectionState.waiting) {
            return const Center(child: CircularProgressIndicator());
          }
          if (snapshot.hasError) {
            return Center(child: Text('Error: ${snapshot.error}'));
          }

          final posts = snapshot.data!;
          // RefreshIndicator adds pull-to-refresh support around the list
          return RefreshIndicator(
            onRefresh: _refresh,
            child: ListView.builder(
              itemCount: posts.length,
              itemBuilder: (context, index) {
                final post = posts[index];
                // Each post is shown inside a Card for a clean, readable look
                return Card(
                  margin: const EdgeInsets.symmetric(horizontal: 12, vertical: 6),
                  elevation: 2,
                  child: Padding(
                    padding: const EdgeInsets.all(12),
                    child: Column(
                      crossAxisAlignment: CrossAxisAlignment.start,
                      children: [
                        Text(
                          post.title,
                          style: const TextStyle(fontWeight: FontWeight.bold, fontSize: 16),
                        ),
                        const SizedBox(height: 6),
                        Text(post.body, maxLines: 2, overflow: TextOverflow.ellipsis),
                      ],
                    ),
                  ),
                );
              },
            ),
          );
        },
      ),
      // A floating button confirming the whole lab manual is complete
      floatingActionButton: FloatingActionButton.extended(
        onPressed: () {
          ScaffoldMessenger.of(context).showSnackBar(
            const SnackBar(content: Text('🎉 Congratulations! Lab manual completed!')),
          );
        },
        label: const Text('Finish'),
        icon: const Icon(Icons.celebration),
      ),
    );
  }
}

Expected Output

Ten neatly styled cards appear, each showing a post's bold title and a two-line preview of its body. Pulling down from the top re-fetches fresh data with a spinning refresh indicator. Tapping the "Finish" button at the bottom right shows a final congratulatory SnackBar.
