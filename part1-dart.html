PART I — THE DART LANGUAGE
Foundations you need before Flutter makes sense

Unit 1Introduction to Dart

Dart is a client-optimized, object-oriented, garbage-collected programming language created by Google and first released in 2011. It was redesigned significantly for mobile and web app development, and today it exists primarily as the language that powers Flutter. Dart compiles in two directions: to native ARM/x64 machine code (used for fast-starting, high-performance mobile and desktop apps) and to JavaScript (used for the web). This dual compilation, combined with a Just-In-Time (JIT) compiler for development and an Ahead-Of-Time (AOT) compiler for release builds, is what gives Flutter apps both a fast development cycle (hot reload) and near-native runtime performance.

Why Dart for Flutter?

  • Fast development cycles — Dart's JIT compiler enables Flutter's *hot reload*, which injects updated code into a running app in under a second without losing app state.
  • Fast, predictable performance — Dart's AOT compiler produces native ARM/x64 code, so release apps do not carry a JavaScript-bridge performance tax the way some cross-platform frameworks do.
  • Everything is an object — including numbers, functions, and null. This gives Dart a small, consistent set of rules with very few special cases.
  • Sound null safety — the compiler statically guarantees that a variable declared as non-nullable can never hold null, eliminating an entire category of runtime crashes.
  • Familiar C-style syntax — curly braces, semicolons, and a syntax family close to Java, C#, and JavaScript make Dart quick to pick up for most programmers.

Setting Up Dart

Dart ships bundled with the Flutter SDK, so installing Flutter also installs Dart. If you only want to experiment with the language itself (without Flutter), you can install the standalone Dart SDK from dart.dev, or write and run snippets instantly in DartPad (dartpad.dev) — a free, browser-based editor that requires no installation and is the fastest way to try the examples in this manual.

To check that Dart is available from your terminal once the SDK is installed:

dart --version
# Expected output looks like:
# Dart SDK version: 3.5.0 (stable) ...

Your First Dart Program

Every Dart application begins execution at a top-level function named main(). Save the following as hello.dart:

void main() {
  print('Hello, World!');
}

Run it from the terminal with:

dart run hello.dart
TIPprint() is Dart's built-in function for writing a line of text to the console. It accepts any object and calls toString() on it automatically, so print(42) and print([1, 2, 3]) both work without extra conversion.
KEY POINTS
  • Dart is the language behind Flutter, designed by Google for fast development and fast execution.
  • JIT compilation powers hot reload during development; AOT compilation produces optimized native code for release.
  • DartPad lets you run Dart in the browser with zero setup — ideal while working through this manual.
  • Execution always starts at the top-level main() function.

Unit 3Control Flow Statements

Conditional Statements

int marks = 78;

if (marks >= 90) {
  print('Grade: A');
} else if (marks >= 75) {
  print('Grade: B');
} else {
  print('Grade: C');
}

// Conditional (ternary) expression
String result = marks >= 40 ? 'Pass' : 'Fail';

Switch Statements

Dart 3 extended switch to support pattern matching, but the classic form is still the one you will use most often for simple value matching:

String grade = 'B';

switch (grade) {
  case 'A':
    print('Excellent');
    break;
  case 'B':
    print('Good');
    break;
  default:
    print('Needs improvement');
}

// Dart 3 switch expression (more concise, returns a value)
String remark = switch (grade) {
  'A' => 'Excellent',
  'B' => 'Good',
  _   => 'Needs improvement',
};

Loops

// for loop
for (int i = 1; i <= 5; i++) {
  print('Count: $i');
}

// for-in loop — iterating a collection
List<String> subjects = ['AI', 'DS', 'ML'];
for (var subject in subjects) {
  print(subject);
}

// while loop — condition checked before each iteration
int n = 0;
while (n < 3) {
  print('n = $n');
  n++;
}

// do-while loop — body always runs at least once
int m = 0;
do {
  print('m = $m');
  m++;
} while (m < 3);

break, continue & Labeled Loops

outerLoop:
for (int i = 0; i < 3; i++) {
  for (int j = 0; j < 3; j++) {
    if (j == 1) continue;        // skip this inner iteration
    if (i == 2) break outerLoop;  // exit the labeled outer loop
    print('i=$i, j=$j');
  }
}
KEY POINTS
  • Dart's classic switch needs an explicit break per case; the newer switch *expression* (=>) does not.
  • for-in is the idiomatic way to loop over any Iterable (List, Set, etc.).
  • do-while guarantees the loop body executes at least once.
  • Labels let break/continue target an outer loop from within a nested one.

Unit 4Functions

Functions are first-class objects in Dart — they can be stored in variables, passed as arguments, and returned from other functions. This is essential in Flutter, where callbacks such as onPressed are simply functions passed as widget parameters.

Basic Function Syntax

int add(int a, int b) {
  return a + b;
}

// Arrow syntax — for a function body that is a single expression
int square(int x) => x * x;

void main() {
  print(add(2, 3));    // 5
  print(square(4));    // 16
}

Optional Positional Parameters

String greet(String name, [String? title]) {
  if (title != null) return 'Hello, $title $name';
  return 'Hello, $name';
}

print(greet('Sudhakar'));            // Hello, Sudhakar
print(greet('Sudhakar', 'Dr.'));     // Hello, Dr. Sudhakar

Named Parameters

Named parameters are wrapped in {} and are the pattern you will see constantly in Flutter widget constructors. Mark a named parameter required when the caller must supply it.

void printProfile({required String name, int age = 18}) {
  print('$name is $age years old');
}

printProfile(name: 'Anita');            // Anita is 18 years old
printProfile(name: 'Ravi', age: 21);    // Ravi is 21 years old
TIPThis is exactly why Flutter widgets are constructed like Text('Hello', style: TextStyle(fontSize: 20)) — the widget's constructor simply uses named parameters.

Anonymous Functions & Closures

List<int> numbers = [1, 2, 3, 4, 5];

// Anonymous function passed directly to forEach
numbers.forEach((n) {
  print('Number: $n');
});

// A closure — the returned function 'closes over' the counter variable
Function makeCounter() {
  int count = 0;
  return () {
    count++;
    return count;
  };
}

void main() {
  var counter = makeCounter();
  print(counter()); // 1
  print(counter()); // 2
}

Higher-Order Functions

List<int> marks = [55, 90, 40, 72, 88];

var passed = marks.where((m) => m >= 50).toList();  // [55, 90, 72, 88]
var doubled = marks.map((m) => m * 2).toList();     // [110, 180, 80, 144, 176]
var total   = marks.reduce((a, b) => a + b);        // 345
var anyFail = marks.any((m) => m < 50);             // true
KEY POINTS
  • Functions are objects — they can be assigned, passed, and returned like any value.
  • [optional] positional parameters and {named} parameters both make arguments optional; required forces a named parameter to be supplied.
  • Arrow syntax (=>) is shorthand for a function body that returns a single expression.
  • where, map, reduce, and any are the functional-style methods you'll use constantly when transforming lists.

List — an ordered, indexable collection

List<String> subjects = ['AI', 'DS', 'ML'];
subjects.add('DL');                 // ['AI', 'DS', 'ML', 'DL']
subjects.insert(1, 'Stats');         // insert at index 1
subjects.remove('ML');

Set<String> tags = {'flutter', 'dart', 'flutter'}; // duplicate is dropped
print(tags);              // {flutter, dart}

Map<String, int> credits = {
  'Flutter Lab': 2,
  'Explainable AI': 4,

Collection If / For & the Spread Operator

Dart allows control-flow-like syntax directly inside a collection literal — extremely useful when building widget lists conditionally in Flutter.

bool showExtra = true;
List<int> base = [1, 2, 3];

List<int> combined = [
  0,
  ...base,                 // spread operator — unpacks all elements
  if (showExtra) 4,        // collection-if
  for (var i in [5, 6]) i, // collection-for
];
print(combined); // [0, 1, 2, 3, 4, 5, 6]

SetNoNot allowedValue lookup (contains)

KEY POINTS
  • List, Set, and Map are the three core collection types, each with a distinct purpose.
  • The spread operator ... and collection-if/-for let you build a collection declaratively — the same technique you'll use to build widget children lists in Flutter.
  • List.unmodifiable() and the const collection literals create collections that cannot be changed after creation.

Unit 6Object-Oriented Programming in Dart

Everything in Dart — including numbers and functions — is an instance of a class. Understanding classes, constructors, and inheritance is essential before Flutter's widget classes will make sense, since every widget is itself a Dart class.

Classes & Objects

class Student {
  String name;
  int rollNumber;

  // Standard constructor
  Student(this.name, this.rollNumber);

  void display() {
    print('$rollNumber: $name');
  }
}

void main() {
  var s1 = Student('Anita', 101);
  s1.display(); // 101: Anita
}

Named & Factory Constructors

class Point {
  double x, y;

  Point(this.x, this.y);

  // Named constructor — an alternative way to build the same class
  Point.origin() : x = 0, y = 0;

  // Factory constructor — has full control over what gets returned,
  // useful for caching or returning a subtype
  factory Point.fromMap(Map<String, double> map) {
    return Point(map['x']!, map['y']!);
  }
}

var p1 = Point(3, 4);
var p2 = Point.origin();
var p3 = Point.fromMap({'x': 1.0, 'y': 2.0});

Inheritance

class Animal {
  String name;
  Animal(this.name);
  void makeSound() => print('$name makes a sound');
}

class Dog extends Animal {
  Dog(String name) : super(name);

  @override
  void makeSound() => print('$name barks');
}

void main() {
  Animal a = Dog('Tommy');
  a.makeSound(); // Tommy barks
}

Abstract Classes & Interfaces

Dart has no separate interface keyword — every class implicitly defines an interface. Use implements to adopt that interface without inheriting behaviour, or extends an abstract class to inherit both a contract and shared implementation.

abstract class Shape {
  double area(); // no body — subclasses must implement this
}

class Circle extends Shape {
  double radius;
  Circle(this.radius);

  @override
  double area() => 3.14159 * radius * radius;
}

Mixins

A mixin lets you reuse a chunk of functionality across multiple, otherwise unrelated class hierarchies — Dart's answer to "multiple inheritance" without the classic diamond problem. Flutter itself relies heavily on mixins, e.g. SingleTickerProviderStateMixin for animations.

mixin Logger {
  void log(String message) => print('[LOG] $message');
}

class NetworkService with Logger {
  void fetchData() {
    log('Fetching data...');
  }
}

Enums

enum Status { pending, approved, rejected }

// Enhanced enums (Dart 2.17+) can carry fields and methods
enum Priority {
  low(1), medium(2), high(3);
  final int level;
  const Priority(this.level);
}

void main() {
  var s = Status.approved;
  print(s);              // Status.approved
  print(Priority.high.level); // 3
}
KeywordPurpose
extendsInherit implementation from exactly one superclass
implementsAdopt a class's interface without inheriting its implementation (must re-implement every member)
withMix in reusable behaviour from one or more mixins
abstractDeclare a class that cannot be instantiated directly, and may contain unimplemented methods
KEY POINTS
  • Every class defines an implicit interface — implements uses that; extends inherits actual behaviour.
  • Named and factory constructors give you multiple, clearly-labelled ways to build the same class.
  • Mixins (with) let you share behaviour across unrelated class hierarchies — heavily used inside Flutter itself.
  • Enhanced enums can carry fields, constructors, and methods, not just a list of names.

Unit 7Generics

Generics let you write a class or function that works with any type while still catching type errors at compile time, instead of relying on dynamic and discovering mistakes at runtime. Dart's own collection types (List<T>, Map<K,V>) are generic classes, and Flutter's State<T extends StatefulWidget> is a generic class too.

Generic Classes

class Box<T> {
  T value;
  Box(this.value);

  void show() => print('Box contains: $value');
}

void main() {
  var intBox = Box<int>(42);
  var strBox = Box<String>('Flutter');
  intBox.show();  // Box contains: 42
  strBox.show();  // Box contains: Flutter
}

Generic Functions

T firstElement<T>(List<T> items) {
  return items[0];
}

print(firstElement<String>(['a', 'b', 'c'])); // a
print(firstElement<int>([10, 20, 30]));       // 10

Bounded Type Parameters

// T must be a num or a subtype of num (int, double)
T largest<T extends num>(T a, T b) => a > b ? a : b;

print(largest<int>(3, 7));       // 7
print(largest<double>(2.5, 1.1)); // 2.5
KEY POINTS
  • Generics give you type safety without duplicating code for every possible type.
  • <T extends SomeType> restricts the generic type to SomeType or its subtypes.
  • You'll see generics constantly in Flutter: List<Widget>, Future<String>, State<MyWidget>.

Unit 8Exception Handling

Dart uses try/catch/finally, similar to Java or C#. All Dart exceptions are unchecked — the compiler does not force you to catch them — so disciplined use of try/catch around risky operations (network calls, parsing, file access) is a habit worth building early.

Basic try-catch

void divide(int a, int b) {
  try {
    print(a ~/ b);
  } on IntegerDivisionByZeroException {
    print('Cannot divide by zero!');
  } catch (e) {
    print('Something went wrong: $e');
  } finally {
    print('Division attempt finished.');
  }
}

void main() {
  divide(10, 0);
}

Custom Exceptions

class InvalidMarksException implements Exception {
  final String message;
  InvalidMarksException(this.message);

  @override
  String toString() => 'InvalidMarksException: $message';
}

void setMarks(int marks) {
  if (marks < 0 || marks > 100) {
    throw InvalidMarksException('Marks must be between 0 and 100');
  }
  print('Marks set to $marks');
}

void main() {
  try {
    setMarks(150);
  } on InvalidMarksException catch (e) {
    print(e);
  }
}
CAUTIONCatching a generic catch (e) swallows every possible error, including bugs you'd rather discover during testing. Prefer on SpecificException catch (e) where you can, and reserve the generic catch for a final safety net.
KEY POINTS
  • on ExceptionType catches a specific exception; a bare catch (e) catches anything.
  • finally always runs, whether or not an exception was thrown — ideal for closing streams or resetting UI loading state.
  • Implementing the Exception interface (implements Exception) is the idiomatic way to define your own exception types.

Unit 9Asynchronous Programming — Future, async/await & Stream

Almost everything meaningful an app does — reading a file, calling an API, querying a database — takes time and should not block the UI thread while it happens. Dart handles this with a single-threaded event loop plus two core types: `Future` (a value that will be available *once*, later) and `Stream` (a sequence of values delivered *over time*).

Future & async/await

Future<String> fetchUserName() async {
  await Future.delayed(Duration(seconds: 2)); // simulate network delay
  return 'Sudhakar';
}

void main() async {
  print('Fetching...');
  String name = await fetchUserName();
  print('Got: $name');
}
// Output:
// Fetching...
// (2 second pause)
// Got: Sudhakar

A function marked async always returns a Future, even if you write a plain return value; inside it — Dart wraps the value for you. await pauses execution *of that function only* until the Future completes; it does not block the rest of the app.

Handling Errors in async Code

Future<int> riskyOperation() async {
  throw Exception('Network error');
}

void main() async {
  try {
    await riskyOperation();
  } catch (e) {
    print('Caught: $e');
  }
}

Chaining Futures with .then()

fetchUserName()
  .then((name) => print('Hello, $name'))
  .catchError((e) => print('Error: $e'))
  .whenComplete(() => print('Done'));
TIPasync/await and .then() chains are two syntaxes for the same underlying mechanism. Prefer async/await for readability — it reads top-to-bottom like synchronous code — and reach for .then() mainly when you need to fire-and-forget without pausing the current function.

Stream — a sequence of asynchronous values

Stream<int> countStream() async* {
  for (int i = 1; i <= 5; i++) {
    await Future.delayed(Duration(seconds: 1));
    yield i;
  }
}

void main() async {
  await for (var value in countStream()) {
    print('Received: $value');
  }
  print('Stream finished');
}
ConceptDeliversTypical Flutter use
Future<T>A single value, once, laterOne-off API call, reading a file
Stream<T>Zero or more values, over timeLive text field changes, StreamBuilder, real-time database listeners
async* / yieldBuilds a Stream incrementallyGenerating a sequence of values lazily
KEY POINTS
  • Future = one value later; Stream = many values over time.
  • async + await is the readable way to work with Futures; the function must be marked async to use await inside it.
  • await for iterates a Stream the same way for-in iterates a List.
  • Always wrap await calls that can fail in try/catch — an uncaught exception inside an async function completes its Future with an error rather than crashing immediately, which is easy to miss.

Unit 10Null Safety

Since Dart 2.12, sound null safety is the default for all new code. The compiler tracks, for every variable, whether null is a legal value — and refuses to compile code where a non-nullable variable might end up null. This eliminates the classic "null pointer / NoSuchMethodError on null" crash at compile time rather than runtime.

Nullable vs Non-Nullable Types

String name = 'Sudhakar';   // non-nullable — can never be null
String? nickname;           // nullable — defaults to null, marked with '?'

// name = null;   // Compile-time error — not allowed
nickname = null;  // OK — declared nullable

Null-Aware Operators

OperatorMeaningExample
?.Call a member only if the object isn't nulluser?.name
??Fallback value if the expression is nullnickname ?? 'Guest'
??=Assign only if the variable is currently nullnickname ??= 'Guest'
!Assert to the compiler "this is not null" (throws at runtime if wrong)nickname!.length
String? nickname;
print(nickname?.toUpperCase());     // null (safe — no crash)
print(nickname ?? 'Guest');          // Guest
nickname ??= 'Ravi';
print(nickname);                     // Ravi
print(nickname!.toUpperCase());      // RAVI — safe now that we know it's set
CAUTIONThe ! (null-assertion) operator is a promise you're making to the compiler, not a safety check. If the value actually is null at runtime, ! throws immediately. Use it only when you are certain — e.g. right after a null check — and prefer ?? or ?. wherever a sensible fallback exists.

The late Keyword

late tells the compiler "trust me, this non-nullable variable will be assigned before it's used" — useful for values initialized in initState() in a Flutter StatefulWidget, where the field must be declared before it's set.

class UserSession {
  late String token; // not assigned yet, but promised to be non-null before use

  void login(String t) {
    token = t;
  }
}
KEY POINTS
  • A type followed by ? is nullable; without ?, the compiler guarantees the variable is never null.
  • ??, ?., and ??= let you handle possible null values gracefully.
  • ! forcibly asserts non-null and will throw at runtime if you're wrong — use sparingly.
  • late defers a non-nullable variable's initialization, common for fields set inside initState().

Unit 11Libraries, Packages & pub.dev

Every Dart file is implicitly its own library. You share code between files with import, and share code between *projects* by publishing or consuming packages through pub.dev, the official Dart/Flutter package repository.

Importing Files

// core Dart library
import 'dart:math';

// a third-party package (declared in pubspec.yaml)
import 'package:http/http.dart' as http;

// a file from within your own project
import 'models/student.dart';

void main() {
  print(sqrt(16)); // 4.0 — from dart:math
}

pubspec.yaml — the project manifest

Every Dart/Flutter project has a pubspec.yaml file at its root declaring the project's name, SDK constraints, and dependencies.

name: my_app
description: A sample Flutter application.
version: 1.0.0+1

environment:
  sdk: '>=3.0.0 <4.0.0'

dependencies:
  flutter:
    sdk: flutter
  http: ^1.2.0
  provider: ^6.1.0

dev_dependencies:
  flutter_test:
    sdk: flutter

After editing pubspec.yaml, fetch the declared packages with:

flutter pub get
TIPThe caret (^1.2.0) means "any version compatible with 1.2.0" — i.e. >=1.2.0 <2.0.0. This lets your project automatically receive bug-fix and minor updates without risking a breaking major-version change.

Organizing Your Own Code into Libraries

// file: utils/validators.dart
bool isValidEmail(String email) => email.contains('@');

// file: main.dart
import 'utils/validators.dart';

void main() {
  print(isValidEmail('test@lbrce.ac.in')); // true
}
KEY POINTS
  • import 'dart:...' brings in core SDK libraries; import 'package:...' brings in third-party or your own package's code.
  • pubspec.yaml declares every dependency your project needs; flutter pub get downloads them.
  • Caret syntax (^x.y.z) in version constraints allows safe, non-breaking updates automatically.

Unit 12Extension Methods

Extension methods let you add new functionality to an existing type — including types you don't own, like String or int from the Dart core library — without subclassing it or modifying its source. This is purely a compile-time, syntactic convenience; it does not change the underlying class.

Defining and Using an Extension

extension StringValidation on String {
  bool get isValidEmail => contains('@') && contains('.');

  String capitalize() {
    if (isEmpty) return this;
    return this[0].toUpperCase() + substring(1);
  }
}

void main() {
  print('sudhakar@lbrce.ac.in'.isValidEmail); // true
  print('flutter'.capitalize());              // Flutter
}

Extensions on Custom and Generic Types

extension ListSum on List<int> {
  int get total => fold(0, (sum, item) => sum + item);
}

void main() {
  print([10, 20, 30].total); // 60
}
TIPExtension methods are resolved statically, based on the variable's declared type at the call site — not its runtime type. If two imported extensions define the same member name for the same type, you'll need to disambiguate with an explicit import 'package:x.dart' show ExtensionName or by calling ExtensionName(value).member.
KEY POINTS
  • Extensions add methods/getters to existing types without inheritance or modifying the original source.
  • They are resolved at compile time based on static type, not the runtime object's actual type.
  • Widely used in Flutter codebases for small conveniences, e.g. context.theme, .toDateString() on DateTime.

Unit 13Records & Pattern Matching (Dart 3)

Dart 3 introduced two closely related features that significantly reduce boilerplate: records, a way to bundle multiple values together without declaring a class, and pattern matching, a way to inspect and destructure data in a single, declarative expression.

Records

// A record with positional fields
(String, int) getStudent() {
  return ('Anita', 101);
}

// A record with named fields — often clearer
({String name, int rollNumber}) getStudentNamed() {
  return (name: 'Anita', rollNumber: 101);
}

void main() {
  var (name, roll) = getStudent();     // destructuring assignment
  print('$name: $roll');

  var student = getStudentNamed();
  print('${student.name}: ${student.rollNumber}');
}
TIPRecords are the idiomatic replacement for returning multiple values from a function — previously you'd need a small custom class or an untyped List/Map just to return a pair of values.

Pattern Matching with switch

String describe(Object value) {
  return switch (value) {
    int n when n < 0 => 'Negative number',
    int n when n == 0 => 'Zero',
    int() => 'Positive number',
    String s => 'A string of length ${s.length}',
    [int a, int b] => 'A pair: $a, $b',
    _ => 'Unknown type',
  };
}

void main() {
  print(describe(-5));       // Negative number
  print(describe('hi'));     // A string of length 2
  print(describe([1, 2]));   // A pair: 1, 2
}

Destructuring in Practice

Map<String, dynamic> json = {'name': 'Ravi', 'age': 21};

// Destructure a map directly in a variable declaration
var {'name': String name, 'age': int age} = json;
print('$name is $age');
KEY POINTS
  • Records ((...) or (field: value, ...)) bundle multiple values without needing a dedicated class.
  • Pattern matching in switch expressions can match on type, structure, and conditions (when) in one place.
  • Destructuring assignment pulls values straight out of records, lists, and maps into named variables.

Unit 14Isolates & Concurrency

Dart code runs on a single thread by default, using an event loop to interleave asynchronous work (this is what Future/async/await manage). For genuinely CPU-intensive work — parsing a huge JSON file, image processing, complex calculations — that would otherwise block the UI thread and cause visible jank, Dart provides isolates: independent workers with their own memory, running in true parallel.

Why Not Just Use a Future?

A Future schedules work to run *later* on the same single thread — it does not run in parallel, and a long, synchronous, CPU-bound loop inside an async function will still freeze the UI, because the event loop can't process anything else until that loop finishes. Isolates solve this by running the heavy work on a genuinely separate thread.

compute() — the simple way in Flutter

Flutter's compute() function wraps isolate creation for the common case: run one function, with one argument, on a background isolate, and get the result back as a Future.

import 'package:flutter/foundation.dart';

// This function must be a top-level or static function
int expensiveCalculation(int input) {
  int result = 0;
  for (int i = 0; i < 1000000000; i++) {
    result += i % input;
  }
  return result;
}

Future<void> runInBackground() async {
  final result = await compute(expensiveCalculation, 7);
  print('Result: $result');
}

Manual Isolates with Isolate.spawn

import 'dart:isolate';

void heavyWork(SendPort sendPort) {
  int sum = 0;
  for (int i = 0; i < 1000000000; i++) {
    sum += i;
  }
  sendPort.send(sum);
}

Future<void> main() async {
  final receivePort = ReceivePort();
  await Isolate.spawn(heavyWork, receivePort.sendPort);
  final result = await receivePort.first;
  print('Sum: $result');
}
CAUTIONIsolates do not share memory — data passed between them is copied, not referenced. This means you cannot pass things like an open database connection or a BuildContext into an isolate; only simple, serializable data (numbers, strings, lists, maps of those) can cross the boundary safely.
KEY POINTS
  • Dart's event loop handles asynchronous *waiting* (Futures) on a single thread; it does not give you true parallelism.
  • Isolates run on separate threads with their own memory, ideal for genuinely CPU-heavy work.
  • compute() is the easiest entry point for one-off background computation in a Flutter app.

Unit 16Iterables — A Functional Programming Deep Dive

Unit 4 introduced where, map, and reduce. Dart's Iterable interface (implemented by List, Set, and more) offers a much richer functional toolkit that, once familiar, replaces the majority of manual loops you'd otherwise write.

MethodReturnsPurpose
map(f)IterableTransform every element
where(f)IterableKeep only elements matching a condition
fold(initial, f)A single valueAccumulate a result across all elements, with a starting value
reduce(f)A single valueLike fold, but uses the first element as the starting value (fails on empty)
expand(f)IterableMap each element to zero or more results and flatten them
take(n) / skip(n)IterableFirst n elements / all but the first n
takeWhile(f) / skipWhile(f)IterableElements while a condition holds, from the start
firstWhere(f)A single elementFirst element matching a condition (throws if none found — use orElse)
every(f) / any(f)boolTrue if all / any elements satisfy the condition
toList() / toSet()List / SetMaterialize a lazy iterable into a concrete collection
List<Map<String, dynamic>> students = [
  {'name': 'Anita', 'marks': 92},
  {'name': 'Ravi', 'marks': 58},
  {'name': 'Priya', 'marks': 76},
];

// fold — compute the average marks
double average = students.fold(0, (sum, s) => sum + (s['marks'] as int)) / students.length;
print(average); // 75.33...

// where + map chained — names of students who passed with distinction
var distinctionNames = students
    .where((s) => (s['marks'] as int) >= 75)
    .map((s) => s['name'])
    .toList();
print(distinctionNames); // [Anita, Priya]

// expand — flatten a list of lists
var nested = [[1, 2], [3, 4], [5]];
var flat = nested.expand((list) => list).toList();
print(flat); // [1, 2, 3, 4, 5]

// firstWhere with a fallback
var topper = students.firstWhere(
  (s) => (s['marks'] as int) > 90,
  orElse: () => {'name': 'None', 'marks': 0},
);
print(topper['name']); // Anita
TIPIterable operations are lazy by default — chaining .where().map() does not actually iterate the collection until something forces evaluation, like .toList(), .length, or a for-in loop. This means you can chain several transformations without worrying about intermediate copies being built at each step.
KEY POINTS
  • fold and reduce both accumulate a single result; fold lets you specify a starting value, reduce does not.
  • expand flattens one level of nesting while mapping — useful for turning a list of lists into a single list.
  • Iterable chains are lazy; nothing actually runs until you call something that forces evaluation.

Creating and Reading Dates

CAUTIONDateTime.now() returns the device's local time by default. When storing timestamps that need to be compared across time zones (e.g. a server-side deadline), convert explicitly with .toUtc() before storing, and .toLocal() only when displaying to the user.
KEY POINTS
  • DateTime represents a specific point in time; Duration represents a span of time.

  • Store timestamps in UTC and convert to local time only for display, to avoid time-zone bugs.
