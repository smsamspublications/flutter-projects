Unit CCommon Errors & Troubleshooting

Error / SymptomLikely CauseFix
RenderFlex overflowed by N pixelsA Row/Column's children need more space than is availableWrap the offending child in Expanded/Flexible, or make the parent scrollable
setState() called after dispose()An async callback (timer, network response) fired after the widget was removed from the treeCheck if (mounted) before calling setState() inside async callbacks
type 'Null' is not a subtype of type 'X'A nullable value was used where a non-null value was expectedAdd a null check, use ??, or fix the source of the null value
The method 'X' was called on nullPre–null-safety style bug — accessing a member on a null objectEnsure null safety is enabled (default since Dart 2.12) and check nullability
"Hot reload" doesn't reflect a changeChange affects main(), global/static state, or an enum — these require a full restartPerform a hot restart (R) instead of hot reload
No MaterialLocalizations foundA Material widget (e.g. Text, Tooltip) was used above/outside a MaterialAppEnsure the widget is inside MaterialApp's widget tree
Vertical viewport was given unbounded heightA ListView/Column of unknown height was placed inside another unbounded-height parentGive the parent a fixed height, or wrap the list in Expanded/SizedBox
Duplicate GlobalKey detectedThe same GlobalKey was assigned to more than one widgetEnsure each GlobalKey is unique, typically one per stateful form/widget instance
Gradle build fails after adding a pluginAndroid Gradle/Kotlin/plugin version mismatchRun flutter clean, then flutter pub get, and check the plugin's required Android Gradle Plugin version
App builds but assets don't showAsset not declared in pubspec.yaml, or a typo in the pathDouble-check indentation and paths under flutter: assets: and re-run flutter pub get
TIPflutter clean followed by flutter pub get resolves a surprising number of "it worked yesterday" build issues by clearing stale cached build artifacts.

Unit GGlossary of Terms

TermMeaning
WidgetAn immutable description of a piece of UI
StateMutable data associated with a StatefulWidget, held in a separate State object
BuildContextA handle to a widget's location in the widget tree, used to look up inherited data
Hot reloadInjecting updated code into a running app without losing state
Hot restartRestarting the app from scratch, resetting all state, faster than a full stop/run
Widget treeThe declarative hierarchy of widgets describing the UI
Element treeThe live, mutable instantiation of the widget tree managing lifecycle
RenderObjectThe layer responsible for layout, painting, and hit-testing
FutureA value (or error) that will be available once, at some point in the future
StreamA sequence of asynchronous values delivered over time
MixinReusable functionality that can be "mixed in" to otherwise unrelated classes via with
Null safetyCompile-time guarantee that non-nullable variables can never hold null
pubspec.yamlThe manifest file declaring a project's dependencies, assets, and metadata
ScaffoldA widget providing the standard visual structure of a screen (app bar, body, drawer, etc.)
ChangeNotifierA class that can notify listeners when its internal data changes, the basis of Provider
Ticker / vsyncMechanism synchronizing animations with the device's screen refresh rate
AOT / JIT compilationAhead-of-time (release, optimized) vs Just-in-time (development, supports hot reload) compilation

Unit HPractice & Viva Questions

Short-answer questions organized by part, suitable for self-assessment, tutorial sheets, or viva-voce preparation. Answers can be located in the corresponding unit referenced.

Part I — Dart Language

  1. What is the difference between final and const? Give an example where only final would work. (Unit 2)
  2. Why does Dart use ~/ instead of a truncating / for integer division? (Unit 2)
  3. What is the difference between a classic switch statement and a Dart 3 switch *expression*? (Unit 3)
  4. Explain the difference between optional positional parameters ([...]) and named parameters ({...}). (Unit 4)
  5. What does it mean for a function to be a "closure"? Give an example. (Unit 4)
  6. Compare List, Set, and Map in terms of ordering and duplicate handling. (Unit 5)
  7. Why does Dart not need a separate interface keyword? (Unit 6)
  8. What is a mixin, and how is it different from inheritance via extends? (Unit 6)
  9. Why are generics preferable to using dynamic for a container class? (Unit 7)
  10. What is the difference between on ExceptionType catch (e) and a bare catch (e)? (Unit 8)
  11. Explain the difference between Future and Stream with one real-world example of each. (Unit 9)
  12. What problem does sound null safety solve, and what does the ! operator actually do at runtime? (Unit 10)
  13. What is the purpose of pubspec.yaml, and what does the caret (^) mean in a version constraint? (Unit 11)

Part II — Flutter Framework

  1. Explain the relationship between the widget tree, element tree, and render tree. (Unit 12)
  2. What is the difference between StatelessWidget and StatefulWidget? When would you choose one over the other? (Unit 15)
  3. Why is calling setState() necessary — what would happen if you simply mutated a state field directly? (Unit 15)
  4. What is the purpose of Expanded and Flexible inside a Row/Column? (Unit 17)
  5. Why should ListView.builder be preferred over ListView(children: [...]) for long lists? (Unit 17)
  6. Describe the flow of data when using Navigator.push with await and a Navigator.pop(context, result) on the pushed screen. (Unit 19)
  7. What problem does Provider/ChangeNotifier solve that setState() alone cannot? (Unit 20)
  8. Explain the difference between context.watch<T>() and context.read<T>(). (Unit 20)
  9. Why is a fromJson factory constructor a better practice than passing raw Map<String, dynamic> objects around an app? (Unit 21)
  10. When would you choose sqflite over shared_preferences? (Unit 22)
  11. What is the difference between an implicit animation (e.g. AnimatedContainer) and an explicit one using AnimationController? (Unit 23)
  12. Why must an AnimationController be disposed, and where should that happen? (Unit 23)
  13. What is the difference between a unit test, a widget test, and an integration test? (Unit 26)
  14. What is the difference between debug, profile, and release build modes, and when should each be used? (Unit 27)

Advanced Topics & Capstone

  1. How do records improve on returning multiple values from a function compared to pre-Dart-3 approaches?
  2. Why can't a BuildContext or an open database connection be passed into a Dart isolate?
  3. What problem does the repository pattern solve, and how does it make unit testing easier?
  4. In the capstone project, what would need to change to replace local sqflite storage with a remote REST API? What would *not* need to change, and why?
  5. Why does the capstone's DetailsScreen check context.mounted after awaiting a dialog result?
